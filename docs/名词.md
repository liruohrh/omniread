# 各类型内容的具体信息拆解
1. 小说
主体信息：标题、作者、简介 / 描述、创作状态（连载 / 完结 / 断更）、首发平台、发布时间、总字数、标签（题材 / 风格）、点赞数、收藏数、人气值、评分、是否付费（整体）、版权信息
节点信息（章节）：章节标题、章节序号、更新时间、是否付费、字数、是否 VIP 章节、作者备注
节点内容信息：章节正文文本、段落格式标记
2. 漫画
主体信息：标题、作者 / 画师、编剧（如有）、简介 / 描述、创作状态、首发平台、发布时间、总话数、标签、点赞数、人气值、评分、是否付费（整体）、版权信息、画风风格、适配设备（竖屏 / 横屏）
节点信息（话 / 卷）：话 / 卷标题、序号、更新时间、是否付费、页数、是否 VIP、作者留言、是否有彩页
节点内容信息：图片资源（分镜图）、图片分辨率、对白文本、特效文字、分页标记
3. 动漫（动画）
主体信息：标题（含原名 / 译名）、制作公司、导演、编剧、声优阵容、简介 / 剧情梗概、播出状态（连载 / 完结）、首播时间、总集数、标签（题材 / 风格）、评分、播放量、追番数、点赞数、是否付费（整体）、版权信息、集均时长、画质规格（1080P/4K）
节点信息（单集）：集标题、集数、播出时间、是否付费、时长、是否 VIP、片头 / 片尾时长、是否有彩蛋
节点内容信息：视频资源（分辨率 / 格式）、字幕文本、配音音频、背景音乐、画面帧率
4. 电视剧
主体信息：标题（含原名 / 译名）、出品方、导演、编剧、主演阵容、简介、播出状态、首播时间、总集数、标签、评分、播放量、热度值、是否付费、版权信息、集均时长、播出平台
节点信息（单集）：集标题、集数、播出时间、是否付费、时长、是否有广告、剧集预告链接
节点内容信息：视频资源、字幕、音频、画面比例、花絮（如有）
5. 电影
主体信息：标题（含原名 / 译名）、出品方、导演、编剧、主演、简介、上映时间、总时长、标签、评分、票房、热度值、是否付费、版权信息、画质规格、分级（PG-13/PG 等）
节点信息（无分节点，可补充版本信息）：版本类型（院线版 / 导演剪辑版）、上线时间、是否付费、播放平台
节点内容信息：视频资源、字幕、音频（杜比 / 环绕声等）、彩蛋片段（如有）

## 抽象对象
- Content（内容主体）
  - 基础属性：
    - content_id（内容ID）：全局唯一编码，用于App内数据标识和关联
    - content_type（内容类型）：novel/comic/anime/tv_play/movie（小说/漫画/动漫/电视剧/电影）
    - title（标题）：内容主标题，支持多语言时可扩展title_cn/title_en
    - sub_title（副标题/别名）：补充标题，如动漫的“XX 第二季”
    - core_creator（核心创作者）：author/director/illustrator（作者/导演/画师，按类型适配）
    - assistant_creator（辅助创作者）：voice_actor/actor/assistant_illustrator（声优/演员/助理画师）
    - description（简介）：App内展示的核心剧情/内容说明，控制字数适配UI
    - publish_status（发布状态）：serializing/completed/to_be_published（连载中/已完结/待发布）
    - first_publish_time（首发时间）：内容首次上线的时间戳（App内统一用时间戳处理）
    - publish_platform（发布平台）：数组形式，如["XX小说网","XX视频App"]
    - copyright_info（版权信息）：版权方、授权期限、地域限制等
    - tags（标签）：数组形式，如["悬疑","治愈","科幻"]，用于App内分类和推荐
  - 数据属性：
    - score（评分）：0-10分制，整合用户评分和平台评分
    - popularity_metric（热度指标）：popularity/play_count/box_office（人气值/播放量/票房，按需取值）
    - interaction_metric（互动指标）：like_count/comment_count/share_count（点赞数/评论数/分享数）
  - 商业属性：
    - overall_pay_mode（整体付费模式）：free/pay/vip_only（免费/按次付费/仅会员）
    - pricing_rule（定价规则）：如“单章0.5元”“全本30元”“会员免费”，存储规则描述或关联定价ID
  - 格式属性：
    - total_quantity（总数量/总时长）：total_words/total_pages/total_episodes/total_duration（总字数/总页数/总集数/总时长）
    - adapt_spec（适配规格）：resolution/quality/layout（分辨率/画质/排版格式，如“1080P”“竖屏排版”）
  - 内容单元集合（ContentUnitList）：
    - ContentUnit（内容单元）：
      - unit_id（单元ID）：App内唯一标识，如章节ID、单集ID
      - unit_name（单元名称）：章节名/单集名/单话名，如“第1章 初遇”“第5集 抉择”
      - unit_serial_number（单元序号）：纯数字，用于排序，如1/2/3
      - unit_publish_time（单元发布时间）：该单元上线的时间戳
      - unit_pay_mode（单元付费模式）：free/pay/vip_only（可与整体模式不同，如免费小说的付费番外）
      - unit_attach_info（单元附加信息）：author_note/has_color_page/opening_duration（作者备注/是否有彩页/片头时长）
      - unit_quantity（单元数量/时长）：unit_words/unit_pages/unit_duration（单章字数/单话页数/单集时长）
      - unit_content（单元内容）：
        - content_carrier_type（内容载体类型）：text/image/video（文本/图片/视频）
        - raw_resource（原始资源）：文本内容/图片URL数组/视频播放地址（适配App资源加载逻辑）
        - attach_content（附加内容）：subtitle/audio/special_text（字幕/音频/特效文字）

## Rust 实现

基于上述概念，在 Rust 中简化为以下核心结构：

### ContentType (内容类型)

```rust
enum ContentType { Novel, Comic, Anime, Video }
```

### Book (书籍/内容主体)

统一的内容主体，适用于小说、漫画等。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 唯一标识 |
| content_type | ContentType | 内容类型 |
| title | String | 标题 |
| author | String | 作者 |
| cover | Option<String> | 封面 URL |
| description | Option<String> | 简介 |
| status | Option<String> | 状态 (连载/完结) |
| tags | Vec<String> | 标签 |
| url | Option<String> | 详情页 URL |
| extra | HashMap | 扩展字段 (评分、人气等) |

### Chapter (章节)

章节元数据及内容，内容直接存储在章节上。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | 章节 ID |
| title | String | 标题 |
| index | i32 | 序号 |
| url | Option<String> | 内容页 URL |
| date | Option<String> | 发布日期 |
| content | Vec<ContentBlock> | 内容块列表 |
| raw_content | Option<String> | 原始内容 (用于 AI 修复) |
| prev_id | Option<String> | 上一章 ID |
| next_id | Option<String> | 下一章 ID |

### ContentBlock (内容块)

```rust
enum ContentBlock {
    Text { text: String },    // 文本段落
    Image { url: String },    // 图片 URL
}
```

- **小说**: content 为文本段落列表
- **漫画**: content 为图片 URL 列表
- **raw_content**: 保存原始 HTML/文本，用于 AI 净化、修复乱文等

### SearchResult (搜索结果)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | String | ID |
| title | String | 标题 |
| author | Option<String> | 作者 |
| cover | Option<String> | 封面 |
| url | Option<String> | 详情页 URL |

### ExploreSection (探索分组)

| 字段 | 类型 | 说明 |
|------|------|------|
| title | String | 分组标题 |
| items | Vec<SearchResult> | 项目列表 |
| more_url | Option<String> | 查看更多 URL |
